## 📦 Versioning

This project follows **Semantic Versioning (SemVer)** for releases.

### Version Format

```
MAJOR.MINOR.PATCH
```

- **MAJOR** → Breaking changes
- **MINOR** → New features (backward-compatible)
- **PATCH** → Bug fixes

### Where Versioning is Used

Versioning is applied to **GitHub releases**.

Example:

- `v1.0.0` → Initial stable release
- `v1.1.0` → New features added
- `v1.1.1` → Bug fixes

> Releases are published in the GitHub “Releases” section.
