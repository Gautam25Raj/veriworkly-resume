# Monorepo Architecture

VeriWorkly uses an **NPM Workspace Monorepo** to manage its massive scale. 

Instead of a single Next.js monolith doing everything—serving marketing pages, dynamic builder routes, API logic, and PDF rendering—we split our infrastructure strictly by concern. This structure creates significant benefits for both DX (Developer Experience) and Production Scalability.

## The 4 Core Applications

### 1. Resume Builder (\pps/resume-builder\)
- **Port:** 3000
- **Framework:** Next.js 15+ (App Router)
- **Role:** The highly interactive flagship application. 
- **Tech:** Uses **Zustand** + \persist\ for local-first storage, minimizing database calls. React Hook Form drives editor states.

### 2. Node.js API Server (\pps/server\)
- **Port:** 8080
- **Framework:** Express + Node.js 20+
- **Role:** Heavy backend operations. 
- **Tech:** Handles PostgreSQL via **Prisma** and Neon DB. Hosts connection pools. Crucially, runs **Playwright Chromium** to execute memory-intensive PDF generation on a secondary thread, ensuring the frontend never freezes.

### 3. Docs Platform (\pps/docs-platform\)
- **Port:** 3001
- **Framework:** Next.js 15+ / Fumadocs
- **Role:** Static Site documentation and interactive API explorers.
- **Tech:** Keeps heavy OpenAPI specs completely out of the Resume Builder bundle.

### 4. Blog Platform (\pps/blog-platform\)
- **Port:** 3002
- **Framework:** Next.js 15+ / Fumadocs
- **Role:** Fast, SEO-optimized marketing hub.

## 📦 Shared Packages (\packages/ui\)
All three frontends share a single source of truth for design via \@veriworkly/ui\. Because of the monorepo workspace, Button components updated in UI immediately propagate exactly to the Blog, Docs, and Editor, powered seamlessly by **Tailwind CSS 4**.

## Scalability Benefits
If our blog goes viral, caching handles the load without stealing Next.js server cycles from our authenticated Resume Editor users. If a massive amount of users request PDF exports, the Playwright Chromium threads scale the \pps/server\ container entirely independently.
