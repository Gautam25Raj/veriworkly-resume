# Quick Start

VeriWorkly is a full-stack application consisting of a **Next.js frontend** and an **Express.js backend**. Because of its robust architecture (utilizing PostgreSQL, Redis, and background PDF workers), getting it running requires setting up a few moving parts.

To get started, choose the path that best fits your goal:

## Architectural Overview

Before you deploy, it helps to understand how the pieces connect. Whether you run locally or via Docker, the data flow remains the same:

1. **Browser (Client):** The user accesses the Next.js frontend (default port `3000`).
2. **Next.js (Web):** Serves the UI and handles local-first Zustand state.
3. **Express API (Server):** Runs separately (default port `8080`) to handle database syncing, authentication, and GitHub integrations.
4. **PostgreSQL (Database):** We highly recommend [Neon](https://neon.tech) for a serverless PostgreSQL experience.
5. **Redis (Cache & Queue):** Used for rate limiting and managing BullMQ background jobs (like Playwright PDF rendering).
