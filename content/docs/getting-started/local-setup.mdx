# Local Development Setup

This guide will walk you through setting up VeriWorkly on your local machine. This path is intended for developers who want to contribute to the codebase, build new resume templates, or debug issues.

---

## Prerequisites

Before you begin, ensure your machine has the following installed:

- **Node.js 20+** (We recommend using `nvm` or `fnm`)
- **Package Manager:** `npm`, `pnpm`, or `yarn`
- **PostgreSQL:** A valid database connection string (We highly recommend [Neon Serverless Postgres](https://neon.tech))
- **Redis:** Installed locally or running via Docker

---

## Installation Guide

<Steps>
  <Step>
    ### Clone the Repository
    First, pull down the open-source repository from GitHub.

    ```bash
    git clone https://github.com/Gautam25Raj/veriworkly-resume.git
    cd veriworkly-resume
    ```

  </Step>

  <Step>
    ### Install Dependencies
    Because VeriWorkly separates the frontend and backend, you must install dependencies in **both** the root directory and the `server/` directory.

    ```bash
    # Install frontend dependencies
    npm install

    # Install backend dependencies
    cd server && npm install
    ```

  </Step>

  <Step>
    ### Configure Environment Variables
    You need to set up environment variables for both the frontend and the backend. Duplicate the example files provided in the repository.

    ```bash
    # From the root directory:
    cp .env.example .env
    cp server/.env.example server/.env
    ```

    Open both `.env` files and fill in your secure details. **Crucial variables include:**
    * `DATABASE_URL`: Your PostgreSQL connection string.
    * `REDIS_URL`: Your Redis connection string (e.g., `redis://localhost:6379`).
    * `AUTH_SECRET`: A random secure string for Better Auth.
    * `NEXT_PUBLIC_BACKEND_URL`: Usually `http://localhost:8080/api/v1`.

  </Step>

  <Step>
    ### Ensure Redis is Running
    VeriWorkly relies heavily on Redis for rate-limiting and background jobs. If you don't have it running, the API will fail to boot.

    <CodeBlockTabs defaultValue="docker" groupId="redis-setup">
      <CodeBlockTabsList>
        <CodeBlockTabsTrigger value="docker">Docker (Recommended)</CodeBlockTabsTrigger>
        <CodeBlockTabsTrigger value="mac">macOS (Homebrew)</CodeBlockTabsTrigger>
        <CodeBlockTabsTrigger value="ubuntu">Ubuntu / Debian</CodeBlockTabsTrigger>
      </CodeBlockTabsList>

      <CodeBlockTab value="docker">

        ```bash
        docker run -d -p 6379:6379 --name redis redis:7
        ```

      </CodeBlockTab>

      <CodeBlockTab value="mac">

        ```bash
        brew install redis
        brew services start redis
        ```

      </CodeBlockTab>

      <CodeBlockTab value="ubuntu">

        ```bash
        sudo apt update
        sudo apt install redis-server
        sudo systemctl start redis
        ```

      </CodeBlockTab>
    </CodeBlockTabs>

  </Step>

  <Step>
    ### Initialize the Database
    Push the Prisma schema to your PostgreSQL database. This creates all the necessary tables for Users, Profiles, Resumes, and Roadmap features.

    ```bash
    # Ensure you are in the server/ directory
    cd server

    # Push the schema to your DB
    npm run db:push
    ```

  </Step>

  <Step>
    ### Boot the Servers
    You will need **two terminal windows** to run the application locally.

    **Terminal 1: The Backend API**
    ```bash
    cd server
    npm run dev
    ```

    **Terminal 2: The Frontend Next.js App**
    ```bash
    # From the root directory
    npm run dev
    ```

  </Step>
</Steps>

## Verification

To ensure everything is working perfectly:

1. Open your browser and navigate to `http://localhost:3000` to see the UI.
2. Navigate to `http://localhost:8080/api/v1/health`. You should see a JSON response stating that the Database and Redis are both `connected`.
