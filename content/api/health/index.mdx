## Description

The Health module provides endpoints to verify that the VeriWorkly API and its underlying infrastructure are operational.

Because VeriWorkly utilizes a decoupled architecture—relying on relational data, in-memory caching, and background workers—a simple HTTP `200` ping is insufficient. The health check endpoint actively probes the internal network to ensure all critical sub-systems are communicating.

---

## Infrastructure Probes

When you hit the health endpoint, the server executes a connection test against:

1. **The Express API Server** (Verifies the Node.js event loop is healthy).
2. **The PostgreSQL Database** (Verifies Prisma can execute a lightweight query).
3. **The Redis Instance** (Verifies the BullMQ queue is ready to accept PDF generation jobs).

---

## Uptime Monitoring & CI/CD

This module is designed to be consumed by automated uptime monitors (like Datadog, UptimeRobot, or Better Stack) and CI/CD pipelines.

To make automated alerting reliable, the endpoint strictly adheres to HTTP status codes to reflect system degradation:

| Consumer              | Purpose                                                                    |
| :-------------------- | :------------------------------------------------------------------------- |
| **Cloud Hosting**     | Kubernetes Liveness/Readiness probes to determine when to restart the pod. |
| **CI/CD Pipelines**   | Pre-deployment verification to ensure the database connection is healthy.  |
| **External Monitors** | Uptime tracking and SLA violation alerts.                                  |

| Status Code               | Infrastructure State                                        | Action Required                                      |
| :------------------------ | :---------------------------------------------------------- | :--------------------------------------------------- |
| `200 OK`                  | All systems nominal (`ok`).                                 | None.                                                |
| `503 Service Unavailable` | One or more core services (Database/Redis) are unreachable. | **Immediate alert.** The API cannot process resumes. |

<Callout type="warn">
  **Privacy Note:** The health endpoint does not expose any sensitive user data.
  It is safe to expose publicly for monitoring purposes.
</Callout>
